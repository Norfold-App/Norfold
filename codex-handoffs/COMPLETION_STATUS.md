## Discovered gaps

- `EmbedBlock` has no persisted display size/layout field, although Part 0 E1 explicitly requires image **and embed** resize persistence.
- `QuoteBlock` can contain child blocks, but `RenderBlock` flattens a quote through `plainText()`; nested quote structure/depth is therefore lost visually even before adding depth controls.
- Per-block Render/Show-source persistence has no model field yet. Its addition must remain backward-compatible with existing schema-28 serialized block payloads (`encodeDefaults = true`, `ignoreUnknownKeys = false`).
- Native inline links are colored and underlined without URL annotations or a URI callback, so the editor cannot satisfy clickable-link or AppFlowy-style link-editing acceptance through the current `AnnotatedString` path.
